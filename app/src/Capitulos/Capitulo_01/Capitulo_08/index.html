Criar um projeto do React do zero

Neste módulo, usamos um projeto inicial para começar a trabalhar rapidamente. Com essa configuração, foi possível focar no React e em algumas das novas sintaxes. Fique à vontade para usar o projeto inicial em seu trabalho.

Você pode querer criar um projeto do zero. Para começar em uma pasta vazia, siga as etapas desta unidade. As etapas usam o Snowpack, exatamente como no projeto inicial.

Esta unidade é opcional. Se você não precisa criar seu próprio projeto, prossiga para a próxima unidade.

Entender a estrutura do projeto

Nossa configuração principal tem duas pastas principais que armazenam código:

  • público
    • Contém qualquer HTML, CSS, imagem ou outros arquivos estáticos
    • Armazena os arquivos index.html e index.css
  • src
    • Contém todos os arquivos que precisam ser renderizados
    • Armazena todos os arquivos .jsx

Também criaremos dois arquivos para configurar o aplicativo:

  • package.json: contém a lista de pacotes e scripts para o aplicativo
  • snowpack.config.js: contém opções de configuração para o Snowpack

Precisamos de três pacotes principais para nosso aplicativo:

  • Snowpack: usado para renderizar JSX para HTML e JavaScript
  • React: usado para criar os componentes
  • React-DOM: usado para montar o aplicativo

Criar a estrutura inicial

Em um diretório vazio, comece instalando os componentes necessários usando o npm. Em seguida, configure o Snowpack e adicione os scripts ao arquivo package.json.

  1. Abra um terminal ou uma janela de comando. Em seguida, execute estes comandos para criar o diretório e o arquivo package.json para o npm. No Mac ou no Linux, use mkdir e, no Windows, use md para criar um diretório.
    2. Bash # Windows md react-recipes && cd react-recipes md src md public touch package.json echo "{}" > package.json # Linux or macOS mkdir react-recipes && cd react-recipes mkdir src mkdir public touch package.json echo "{}" > package.json
  2. Execute o código a seguir no mesmo terminal ou janela de comando.
    3. Bash npm install --save-dev snowpack npm install react react-dom

    Observação

    O Snowpack é uma dependência de desenvolvimento. Ou seja, ele não é necessário para a produção, pois ele gera os arquivos JavaScript e HTML requeridos durante o processo de compilação.

  3. Abra o diretório no Visual Studio Code executando o comando a seguir.
    4. Bash code .

Configurar o Snowpack

Uma das vantagens de uma ferramenta como o Snowpack é ser quase totalmente autoconfigurável. No entanto, precisamos indicar a estrutura de pastas do código. Para indicar a estrutura de pastas, definimos as opções no arquivo snowpack.config.json.

  1. No Visual Studio Code, crie um arquivo selecionando Arquivo>Novo Arquivo.
  2. Nomeie o arquivo como snowpack.config.json.
  3. No arquivo novo, adicione o código a seguir.
    4. JavaScript module.exports = { mount: { 'public': '/', 'src': '/dist' } }

Esse código diz ao Snowpack para usar a pasta public como a raiz do aplicativo. Ele também define o diretório src como o local virtual para os arquivos JavaScript e arquivos HTML que ele gerará.

Criar os scripts do npm

Para dar suporte a nosso trabalho de desenvolvimento, usaremos dois scripts com o Snowpack. O primeiro script inicia o servidor de desenvolvimento. Essa ação atualiza automaticamente a página quando modificamos o aplicativo. O segundo script é usado quando estamos prontos para criar todos os nossos arquivos para implantação.

  1. No Visual Studio Code, abra o arquivo package.json.
  2. Na parte inferior do arquivo, acima da última chave (}), adicione o código a seguir. Este código cria os scripts de início e de compilação.
    3. JSON { "scripts": { "start": "snowpack dev", "build": "snowpack build" } }

    O arquivo inteiro deve ser semelhante ao código a seguir.

    JSON { "devDependencies": { "snowpack": "^2.18.5" }, "dependencies": { "react": "^17.0.1", "react-dom": "^17.0.1" }, "scripts": { "start": "snowpack dev", "build": "snowpack build" } }
  3. Salve todos os arquivos selecionando Arquivo>Salvar tudo.

Agora você configurou seu projeto inicial. Você pode adicionar index.html, App.jsx e outros arquivos da mesma forma que fez nas unidades anteriores.
